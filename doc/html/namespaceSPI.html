Detailed Description

Namespace with functions that implement IO requests for SPI bus. Requests are triggered by the SPI slave device object using its read/write functions.

void *  transmit_buffer,
void *  receive_buffer,
int  buffer_size

SPI is full-duplex synchronous bus. While writing data to the slave (content of the transmit buffer), in parallel slave answers with data back to the master (content of the receive buffer). The size of the transmit and receive buffer are the same.

Parameters
[in]spi_mastera file descriptor of the bus master device file
[in]transmit_buffercontent of this buffer will be sent to the slave device
[out]receive_bufferoutput from the device will end up here
[in]buffer_sizesize of the transmit and receive buffers
Returns
0 in case there are no errors on the bus, error code in case of communication failure

int SPI::spi_slave_write_byte ( SPI_Bus_Master_Device *  spi_master,
uint8_t  registry,
uint8_t  value

Sends two bites to the slave device: address of an internal registry of the slave and the new value of this registry. Due to the full-duplex of the SPI bus, the slave will answer to the master with two bits but they will be ignored by this function.

Parameters
spi_masterspi_master a file descriptor of the bus master device file
registrywrite the byte into this registry (at this internal slave address)
valuethis byte will be sent to the slave device
Returns
0 in case there are no errors on the bus, error code in case of communication failure

Writes data to the slave device starting from internal location in the slave device defined with the "registry" argument. Due to the full-duplex of the SPI bus, the slave will answer to the master with the same number of bytes but they will be ignored by this function.

Parameters

buffercontent of this buffer will be sent to the slave device
buffer_sizesize of the transmit buffer
Returns
0 in case there are no errors on the bus, error code in case of communication failure

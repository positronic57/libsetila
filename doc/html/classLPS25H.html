virtual int LPS25H::set_mode_of_operation ( mode_of_operation_t  mode_of_operation,
output_data_rate_t  output_data_rate = ST_Sensor::output_data_rate_t::ODR_ONE_SHOT 
)
overridevirtual

High level function for configuring the sensor. It sets the mode of operation and the output data rate by writing values in the sensors control registers based on the given arguments. The registers and their values for achieving the desired sensor configuration are individual for each ST sensor type. The logic will be implemented in the child class that represents the target ST sensor.

Parameters
[in]mode_of_operationpredefined enum for the modes of operation found in ST sensors
[in]output_data_ratepredefined enum for the output data data rates found in ST sensors
Returns
int 0 in case of successful configuration write, appropriate error code otherwise

Implements ST_Sensor.

Examples:
LPS25H.h, and pi_sense_hat.cpp.

◆ set_mode_of_operation() [2/2]

int LPS25H::set_mode_of_operation ( mode_of_operation_t  mode_of_operation,
output_data_rate_t  output_data_rate,
LPS25H_NBR_AVERAGED_SAMPLES  fifo_mean_samples 
)

Same as set_mode_of_operation() from ST_Sensor class, with additional argument that defines the number of averaged samples in case of FIFO_MEAN mode is selected. Calling set_mode_of_operation() without the last argument for FIFO_MEAN mode will set the number of averaged samples to 2.

Parameters
[in]mode_of_operationdefines the operation mode of the sensor
[in]output_data_ratedefines the output data rate of the sensor
[in]fifo_mean_samplesif FIFO_MEAN mode is given as an value for the argument one, will set the number of averaged samples
Returns
int 0 in case of success, ERROR_WRONG_DEVICE_MODEL/ERROR_WRITE_FAILED in case of a failure
